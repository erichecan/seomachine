## 一、产品概述

### 1.1 产品定位

SEO Machine 是一个 AI 驱动的 SEO 内容创作 SaaS，帮助用户完成从关键词研究到内容发布的完整工作流。

**核心价值主张**：  
- 用 AI 把"写一篇 SEO 文章"的时间从 4 小时压缩到 10 分钟
- 内容质量可量化（SEO 评分、可读性分析、竞品对比）
- 写完即发，直连 WordPress / Shopify / 自定义站点

### 1.2 目标用户

| 用户类型 | 痛点 | 场景 |
|---------|------|------|
| 独立站站长 | 写文章太慢，SEO 知识不足 | 每周需要输出 5-10 篇 SEO 文章 |
| 内容运营 | 批量生产内容效率低 | 管理多个站点的内容更新 |
| SEO 从业者 | 客户需要高频交付 | 为多个客户生成内容并发布 |
| 电商卖家 | 产品页/博客缺乏 SEO 流量 | 为 Shopify 店铺创建博客内容 |

### 1.3 产品边界（MVP）

**在范围内**：
- AI 辅助 SEO 文章生成（多模型路由）
- 内容 SEO 分析（可读性、关键词密度、质量评分）
- WordPress / Shopify / Webhook 一键发布
- 用户注册/登录、套餐订阅（Stripe）
- 用量追踪和 Fair Use 限速
- 品牌声音和 SEO 规范配置

**不在 MVP 范围**（Phase 4）：
- DataForSEO 关键词研究集成
- GA4 / GSC 流量数据集成
- 话题聚类分析
- 竞品内容差距分析
- 内容日历
- 内容再利用（LinkedIn/Medium/Reddit）

## 二、用户故事

### 2.1 新用户注册与引导

```
作为 新用户
我想要 快速注册并完成基础配置
以便  立即开始生成第一篇文章

验收标准：
- 支持邮箱注册和 Google 快速登录
- 注册后进入 Onboarding 流程（5 步内完成）
- Onboarding：配置品牌声音 → 添加 SEO 关键词 → 连接发布渠道（可跳过）→ 选择套餐 → 生成第一篇文章
- 免费试用：新用户可免费生成 3 篇文章，无需信用卡
```

### 2.2 AI 文章写作

```
作为 付费用户
我想要 输入关键词后 AI 自动生成一篇 SEO 优化的文章
以便  节省写作时间

验收标准：
- 输入目标关键词和文章大纲方向（可选）
- 系统根据关键词生成文章结构建议（H2/H3 大纲）
- 用户确认大纲后，AI 生成全文（流式输出，实时展示）
- 生成完成后自动运行 SEO 分析，展示评分和建议
- 生成时间：1500 字文章 < 60 秒，3000 字文章 < 120 秒
```

### 2.3 内容编辑与 SEO 优化

```
作为 用户
我想要 在生成后直接编辑文章内容，并看到实时 SEO 反馈
以便  在发布前确保质量

验收标准：
- Markdown 编辑器，支持基本格式（粗体/标题/列表/链接）
- 右侧 SEO 分析面板（实时，不需要手动触发）
  - 综合 SEO 评分（0-100）
  - 关键词密度（目标：1%-3%）
  - 文章字数 vs 竞品均值对比
  - 可读性评分（Flesch Reading Ease）
  - 缺失的 SEO 元素提示（标题/描述/H1 等）
- 支持手动修改 SEO 元数据（Meta Title / Meta Description / Slug）
- 保存状态自动同步（草稿自动保存）
```

### 2.4 一键发布

```
作为 用户
我想要 把写好的文章直接发布到我的网站
以便  省去复制粘贴的步骤

验收标准：
- 支持 WordPress REST API（应用程序密码认证）
- 支持 Shopify Blog（Partner OAuth 授权）
- 支持自定义 Webhook（URL + API Key）
- 发布前可选择：直接发布 / 保存为草稿
- 发布后显示目标平台的文章链接
- 发布失败时展示明确错误信息（认证失败/网络错误/平台限制）
```

### 2.5 用量和套餐管理

```
作为 用户
我想要 清楚知道我还剩多少生成额度，以及如何升级
以便  合理安排使用计划

验收标准：
- 顶部/侧边栏显示当月已用/剩余篇数
- 接近上限（80%）时发送提醒邮件
- 达到上限后生成按钮禁用，显示"升级以继续"
- 可查看历史每月用量趋势
- 点击"升级"跳转 Stripe Checkout，成功后立即更新套餐
- 支持查看账单历史（Stripe Customer Portal）
- 支持取消订阅（当月到期后降为免费）
```

### 2.6 发布渠道管理

```
作为 用户
我想要 统一管理我的所有发布渠道
以便  轻松切换不同网站发布

验收标准：
- 支持添加多个 WordPress / Shopify / Webhook 渠道
- 每个渠道可自定义名称（如"主站 WordPress"、"英文 Shopify"）
- 提供"测试连接"按钮，验证配置是否正确
- 渠道配置加密存储（不明文存储 API Key）
- 失效渠道（如密码更换）显示警告标识
```

### 2.7 品牌与 SEO 配置

```
作为 用户
我想要 配置我的品牌声音和 SEO 规范
以便  AI 生成的内容风格与我的品牌一致

验收标准：
- 品牌声音：描述语气（正式/轻松/专业）、受众特征、写作风格要求
- SEO 规范：目标关键词列表、内链映射、禁用词列表
- 写作示例：上传 1-3 篇参考文章（AI 学习写作风格）
- 竞品信息：填写主要竞争对手域名
- 配置保存后，所有后续生成自动读取这些配置
```

## 三、功能模块需求

### 3.1 Phase 1：Core SEO Engine

#### 3.1.1 LLM 多模型路由

| 需求 | 描述 |
|------|------|
| 多模型支持 | 集成 DeepSeek V3、GPT-4o mini、GPT-4o |
| 路由逻辑 | 基础版 → DeepSeek；专业版/团队版 → GPT-4o；超量后自动降级 |
| API Key 安全 | LLM API Key 仅存服务端，绝不下发客户端 |
| 流式输出 | 支持 SSE（Server-Sent Events）流式返回，前端实时展示生成内容 |
| 超时处理 | 单次生成超过 120s 返回超时错误，提示用户重试 |
| 错误重试 | LLM API 失败自动重试 1 次，第 2 次失败返回错误 |

#### 3.1.2 文章生成

| 需求 | 描述 |
|------|------|
| 大纲生成 | 根据关键词生成 H2/H3 大纲结构，用户可修改后再生成全文 |
| 全文生成 | 基于大纲生成完整文章，含引言、正文、结论 |
| SEO 元数据生成 | 同时生成 Meta Title（<60 字符）、Meta Description（<160 字符）、URL Slug |
| 品牌配置注入 | 生成时将品牌声音、写作示例、SEO 规范注入 system prompt |
| 字数限制 | 按套餐限制最大字数（基础版 1500 字，专业版 3000 字，团队版无限） |
| 语言支持 | 支持中英文生成（用户在设置中选择默认语言） |

#### 3.1.3 SEO 分析引擎

| 分析维度 | 算法 | 输出 |
|---------|------|------|
| 综合 SEO 评分 | 加权综合评分 | 0-100 分，带各维度分项 |
| 关键词密度 | 词频统计 | 百分比，标注过低/过高/正常 |
| 可读性 | Flesch Reading Ease | 分数 + 年级水平 |
| 内容长度 | 与 SERP TOP10 均值对比 | 字数对比柱状图 |
| 标题结构 | H1/H2/H3 分析 | 层级建议 |
| Meta 信息 | 字符数检查 | 超长/过短提示 |
| 内链机会 | 关键词匹配内链映射 | 建议添加内链列表 |

分析结果在前端编辑器右侧面板实时展示，内容变动后 debounce 500ms 触发重新分析。

#### 3.1.4 发布连接器

**WordPress 连接器**
```
配置项：站点 URL、用户名、应用程序密码
发布接口：POST /wp-json/wp/v2/posts
认证：HTTP Basic Auth（Base64 编码）
支持字段：title、content（HTML）、status、categories、tags
SEO 元数据：_yoast_wpseo_focuskw、_yoast_wpseo_metadesc（Yoast 兼容）
```

**Shopify 连接器**
```
授权流程：Shopify Partner OAuth（用户授权后获取 Access Token）
发布接口：POST /admin/api/2024-01/blogs/{blog_id}/articles.json
支持字段：title、body_html、tags、published
需要：在 Shopify Partner 后台注册 App（Client ID + Secret）
```

**Webhook 连接器**
```
配置项：接口 URL、API Key（Header 名称可自定义）
请求格式：JSON POST
约定结构：
{
  "title": "string",
  "content": "string（Markdown）",
  "seo": {
    "target_keyword": "string",
    "meta_title": "string",
    "meta_description": "string",
    "slug": "string"
  },
  "metadata": {
    "generated_at": "ISO8601",
    "source": "seomachine",
    "word_count": number
  }
}
```

### 3.2 Phase 2：SaaS 后端

#### 3.2.1 用户认证

| 功能 | 说明 |
|------|------|
| 邮箱注册/登录 | 邮箱 + 密码，发送验证邮件 |
| Google OAuth | 快速登录，无需密码 |
| Session 管理 | NextAuth.js / next-auth v5 |
| 密码重置 | 邮箱验证码重置 |
| 账户删除 | GDPR 合规，删除所有用户数据 |

#### 3.2.2 订阅与计费（Stripe）

| 功能 | 说明 |
|------|------|
| 订阅套餐 | 基础版 $29/月、专业版 $79/月、团队版 $199/月 |
| 免费试用 | 新用户 3 篇免费（无需信用卡，无时间限制） |
| Checkout | Stripe Checkout 托管支付页，支持信用卡 |
| Webhook 同步 | 监听 `invoice.payment_succeeded`、`customer.subscription.deleted` 等事件，实时同步套餐状态 |
| Customer Portal | Stripe 托管账单管理页（升降级、查看发票、取消订阅） |
| 年付折扣 | 年付享 2 个月免费（可选，MVP 阶段可先不做） |

#### 3.2.3 Fair Use 中间件

每次生成请求经过以下检查链：

```
1. 验证用户已登录（JWT/Session 检查）
2. 检查套餐是否有效（未过期、未取消）
3. 检查今日请求数（Redis：user:{id}:daily:{date}:count）
   - 基础版：≥ 5 篇 → 返回 429，提示"今日已达上限，明日继续"
   - 专业版：≥ 20 篇 → 同上
   - 团队版：无硬性每日限制
4. 检查每篇字数是否超套餐上限
   - 基础版：> 1500 字 → 截断到 1500 字
   - 专业版：> 3000 字 → 截断到 3000 字
5. 选择模型（根据套餐 + 月度用量）
6. 通过 → 调用 LLM，完成后更新 Redis 计数器
```

Redis Key 设计：
```
user:{userId}:daily:{YYYY-MM-DD}:count     TTL = 25h
user:{userId}:monthly:{YYYY-MM}:count      TTL = 32d
user:{userId}:monthly:{YYYY-MM}:tokens     TTL = 32d
```

#### 3.2.4 多租户数据隔离

- 所有数据表含 `user_id` 外键
- API 路由强制过滤 `WHERE user_id = current_user_id`
- Stripe Customer ID 与用户 1:1 绑定
- 发布渠道的 API Key / Access Token 使用 AES-256-GCM 加密存储（`CHANNEL_ENCRYPTION_KEY` 环境变量）

#### 3.3.1 页面清单

#### 3.3.2 AI 写作流程（核心体验）

```
Step 1 — 输入
  └── 目标关键词（必填）
  └── 文章语言（默认来自账户设置）
  └── 文章风格/角度（可选，如"初学者指南"、"对比测评"）

Step 2 — 大纲确认
  └── AI 生成 H2/H3 大纲（约 5-8 个标题）
  └── 用户可拖拽排序、删除、添加、修改每个标题
  └── 点击"开始生成全文" → 进入 Step 3

Step 3 — 全文生成（流式）
  └── 实时展示生成进度（逐字/逐句流式输出）
  └── 右侧展示"生成中..."状态
  └── 生成完成后自动跳转到编辑器（Step 4）

Step 4 — 编辑与优化
  └── 左侧：Markdown 编辑器
  └── 右侧：SEO 分析面板（评分 + 具体建议）
  └── 底部栏：字数统计 | 可读性 | 关键词密度
  └── 点击"发布" → 弹出渠道选择器（Step 5）

Step 5 — 选择渠道发布
  └── 列出已配置的渠道（带状态图标）
  └── 选择"直接发布"或"保存为草稿"
  └── 发布成功：展示目标平台链接，文章状态更新为"已发布"
```

#### 3.3.3 编辑器要求

| 功能 | 要求 |
|------|------|
| 编辑器库 | 推荐 TipTap（支持 Markdown 快捷键，富文本模式更友好）或 CodeMirror |
| 自动保存 | 内容变更 3 秒后自动保存到数据库（显示"已保存"状态） |
| 版本历史 | MVP 不需要，Phase 4 再做 |
| SEO 面板更新 | debounce 500ms，内容变更后重新计算所有分析维度 |
| 响应式 | 桌面优先（>1024px），移动端可查看但不适合编辑 |

#### 3.3.4 落地页要求

| 区块 | 内容 |
|------|------|
| Hero | 产品标语 + 副标题 + "免费试用"CTA + 截图/演示 GIF |
| 功能特性 | 3-4 个核心功能卡片（关键词→AI写作→SEO分析→一键发布） |
| 演示/截图 | 编辑器截图或演示视频（可后期替换） |
| 定价 | 三档套餐对比表，突出"专业版"推荐 |
| FAQ | 常见问题（如"支持哪些语言"、"是否支持 Shopify"） |
| 页脚 | 链接 + 版权 + 隐私政策 + 服务条款 |

## 四、数据模型

### 用户

```sql
users
  id              uuid PRIMARY KEY
  email           varchar(255) UNIQUE NOT NULL
  name            varchar(255)
  avatar_url      text
  password_hash   varchar(255)      -- Google 登录为 NULL
  email_verified  boolean DEFAULT false
  plan            varchar(20) DEFAULT 'free'  -- free/basic/pro/team
  plan_expires_at timestamptz
  stripe_customer_id varchar(255) UNIQUE
  created_at      timestamptz DEFAULT now()
  updated_at      timestamptz DEFAULT now()
```

### 内容

```sql
articles
  id              uuid PRIMARY KEY
  user_id         uuid REFERENCES users(id) ON DELETE CASCADE
  title           varchar(500) NOT NULL
  content         text                        -- Markdown
  content_html    text                        -- 发布时转换的 HTML
  target_keyword  varchar(255)
  meta_title      varchar(60)
  meta_description varchar(160)
  slug            varchar(255)
  seo_score       integer                     -- 0-100
  word_count      integer
  language        varchar(10) DEFAULT 'zh'
  status          varchar(20) DEFAULT 'draft' -- draft/published/archived
  published_at    timestamptz
  published_channel_id uuid REFERENCES channels(id) ON DELETE SET NULL
  external_post_id varchar(255)               -- 目标平台的文章 ID/URL
  created_at      timestamptz DEFAULT now()
  updated_at      timestamptz DEFAULT now()
```

### 发布渠道

```sql
channels
  id              uuid PRIMARY KEY
  user_id         uuid REFERENCES users(id) ON DELETE CASCADE
  name            varchar(100) NOT NULL
  type            varchar(20) NOT NULL   -- wordpress/shopify/webhook
  config_encrypted text NOT NULL         -- AES-256-GCM 加密的 JSON
  status          varchar(20) DEFAULT 'active'  -- active/error
  last_tested_at  timestamptz
  created_at      timestamptz DEFAULT now()
```

### 用量记录

```sql
usage_records
  id              uuid PRIMARY KEY
  user_id         uuid REFERENCES users(id) ON DELETE CASCADE
  period          varchar(7) NOT NULL    -- YYYY-MM
  articles_count  integer DEFAULT 0
  tokens_input    integer DEFAULT 0
  tokens_output   integer DEFAULT 0
  UNIQUE (user_id, period)
```

### 品牌配置

```sql
brand_configs
  user_id         uuid PRIMARY KEY REFERENCES users(id) ON DELETE CASCADE
  brand_voice     text
  style_guide     text
  seo_guidelines  text
  internal_links  jsonb DEFAULT '[]'
  target_keywords jsonb DEFAULT '[]'
  writing_examples jsonb DEFAULT '[]'   -- [{title, content, notes}]
  competitor_sites jsonb DEFAULT '[]'
  updated_at      timestamptz DEFAULT now()
```

### Stripe 事件日志

```sql
stripe_events
  id              varchar(255) PRIMARY KEY  -- Stripe Event ID
  type            varchar(100)
  processed       boolean DEFAULT false
  payload         jsonb
  created_at      timestamptz DEFAULT now()
```

## 五、API 路由清单

### 认证

```
POST /api/auth/register          邮箱注册
POST /api/auth/login             邮箱登录
GET  /api/auth/[...nextauth]     NextAuth 端点（Google OAuth）
POST /api/auth/forgot-password   发送重置密码邮件
POST /api/auth/reset-password    重置密码
```

### 文章

```
GET    /api/articles             获取文章列表（支持筛选 status、搜索 title）
POST   /api/articles             创建新文章（仅标题和关键词）
GET    /api/articles/[id]        获取单篇文章详情
PATCH  /api/articles/[id]        更新文章内容/状态
DELETE /api/articles/[id]        删除文章（软删除）
```

### AI 生成

```
POST /api/generate/outline       根据关键词生成大纲（非流式）
POST /api/generate/article       根据大纲生成全文（SSE 流式）
POST /api/generate/seo-meta      生成 Meta Title/Description/Slug
POST /api/analyze/seo            对内容执行 SEO 分析（返回评分和建议）
```

### 发布

```
GET    /api/channels             获取渠道列表
POST   /api/channels             添加新渠道
PATCH  /api/channels/[id]        更新渠道配置
DELETE /api/channels/[id]        删除渠道
POST   /api/channels/[id]/test   测试渠道连通性
POST   /api/publish              发布文章到指定渠道
```

GET  /api/billing/subscription   获取当前订阅状态

## 六、非功能性需求

### 性能

| 指标 | 目标 |
|------|------|
| 页面首次加载 | < 3 秒（LCP） |
| API 响应（非生成类） | < 500ms |
| 大纲生成 | < 10 秒 |
| 全文生成首字延迟（TTFB of stream） | < 5 秒 |
| SEO 分析计算 | < 2 秒 |

### 安全

| 要求 | 实现 |
|------|------|
| 所有写操作 API 必须鉴权 | NextAuth Session 验证 |
| LLM API Key 不下发客户端 | 仅存环境变量，服务端调用 |
| 渠道 API Key/Token 加密存储 | AES-256-GCM，`CHANNEL_ENCRYPTION_KEY` |
| Stripe Webhook 签名验证 | 验证 `stripe-signature` Header |
| SQL 注入防护 | Prisma ORM 参数化查询 |
| XSS 防护 | 发布 HTML 前进行 sanitize |
| CSRF 防护 | NextAuth 内置 |

### 可用性与监控

| 要求 | 说明 |
|------|------|
| 错误监控 | 接入 Sentry（免费套餐） |
| 日志 | Vercel 日志 + 关键操作结构化日志 |
| 健康检查 | `GET /api/health` 返回 `{"status":"ok","db":"ok","redis":"ok"}` |
| 限速保护 | 生成 API 限速：每 IP 每分钟最多 10 次请求（防刷） |

### 合规

| 要求 | 实现 |
|------|------|
| 隐私政策 | 落地页提供，GDPR 合规 |
| 数据删除 | 用户删除账户后 30 天内清除所有数据 |
| Cookie | 仅 Session Cookie，无广告追踪 |
| 服务条款 | 含 Fair Use Policy 说明 |
