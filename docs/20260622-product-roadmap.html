决策项 结论 原因
产品形态 只做 SaaS Web 桌面 App 已放弃 避免下载、安装、更新、多系统兼容问题，Web 部署一次所有人可用
计费模式 月度订阅，平台统一承担 LLM 成本 用户无需准备 API Key，降低门槛；后台 Fair Use 控制成本
成本控制 Fair Use：限速 + 模型降级 + 队列延迟 对普通用户无感知，服务条款注明，行业惯例
WordPress 对接 REST API + 应用程序密码 实现简单，主流方案，用户操作门槛低，无需插件
Shopify 对接 Partner App + OAuth Shopify 标准路径，授权体验好，发布到 Blog Posts
自建站对接 通用 Webhook（用户填接口 + Key） 适配任意技术栈，用户控制两端，不依赖任何特定平台
LLM Key 管理 只存服务端，客户端不可见 安全性，同时支持按套餐路由和降级策略
